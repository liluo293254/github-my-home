# 作品集设置清单

## ✅ 部署前检查清单

在部署作品集之前，请完成以下步骤：

### 1. 更新个人信息

- [ ] 修改导航栏品牌名称 (`index.html` 第 20 行)
- [ ] 更新首页标题和描述 (`index.html` 第 35-40 行)
- [ ] 更新"关于我"部分内容 (`index.html` 第 65-75 行)
- [ ] 更新统计数据（项目数、技术栈等）(`index.html` 第 80-95 行)

### 2. 更新项目信息

- [ ] 编辑 `script.js` 中的项目数组
- [ ] 更新项目标题、描述和标签
- [ ] 更新项目演示链接 (`demoUrl`)
- [ ] 更新项目代码链接 (`codeUrl`)
- [ ] 添加或删除项目卡片

### 3. 更新技能标签

- [ ] 在 `index.html` 的 `#skills` 部分更新技能标签
- [ ] 确保只列出您真正掌握的技能
- [ ] 可以添加或删除技能分类

### 4. 更新联系方式

- [ ] 更新 GitHub 链接 (`index.html` 第 165 行)
- [ ] 更新 Email 地址 (`index.html` 第 170 行)
- [ ] 更新 LinkedIn 链接 (`index.html` 第 175 行)
- [ ] 添加其他社交媒体链接（如需要）

### 5. 自定义样式（可选）

- [ ] 修改颜色主题 (`styles.css` 第 1-15 行)
- [ ] 调整字体大小和间距
- [ ] 添加个人品牌元素

### 6. GitHub 仓库设置

- [ ] 创建 GitHub 仓库（如果还没有）
- [ ] 将代码推送到仓库
- [ ] 在仓库设置中启用 GitHub Pages
- [ ] 配置 GitHub Actions（如果使用）

### 7. 测试

- [ ] 在本地浏览器中测试所有链接
- [ ] 检查移动端响应式布局
- [ ] 验证所有图片和资源加载正常
- [ ] 测试表单和交互功能（如果有）

### 8. SEO 优化（可选）

- [ ] 更新 `<meta>` 标签 (`index.html` 第 6-7 行)
- [ ] 添加 Open Graph 标签
- [ ] 添加网站图标 (favicon)

## 🚀 快速部署步骤

1. **完成上述检查清单**

2. **提交代码到 GitHub**:
   ```bash
   git add .
   git commit -m "Add portfolio website"
   git push origin main
   ```

3. **启用 GitHub Pages**:
   - 进入仓库 Settings → Pages
   - 选择 Source: GitHub Actions 或 Deploy from a branch
   - 保存设置

4. **等待部署完成** (通常 1-2 分钟)

5. **访问您的作品集**:
   - `https://yourusername.github.io/repository-name/`
